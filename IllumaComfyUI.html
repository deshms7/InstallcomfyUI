Using ComfyUI

ComfyUI is a node-based interface for AI image generation. Each node performs one step of the pipeline — load a model, encode a prompt, sample, decode, save.

Basic txt2img workflow

Load Checkpoint — in the Load Checkpoint node, click the model dropdown and select a .safetensors file.
Enter your prompt — click the positive CLIP Text Encode node and type your prompt. Enter a negative prompt in the negative node.
Set image size — in the Empty Latent Image node, set width and height (e.g. 512×512 for SD1.5, 1024×1024 for SDXL).
Click Queue Prompt — the blue button at the top right. The GPU begins generating. Progress shows in the top bar.
View output — the generated image appears in the Save Image node and is saved to the outputs folder automatically.

File Locations

All data is stored outside the container and persists across restarts and image updates.

Path on this machinePurpose
/data/comfyui/models/ All model files — checkpoints, LoRAs, VAEs, ControlNets, embeddings
/data/comfyui/models/checkpoints/ Main model files (.safetensors, .ckpt)
/data/comfyui/models/loras/ LoRA / LyCORIS fine-tune weights
/data/comfyui/models/vae/ VAE decoder files
/data/comfyui/models/controlnet/ ControlNet model files
/data/comfyui/outputs/ All generated images land here, organised by date
/data/comfyui/custom_nodes/ Custom node extensions (e.g. ComfyUI-Manager)
ℹ️ These paths are mounted into the container at /opt/ComfyUI/models and /opt/ComfyUI/output. Files you place here are immediately visible to ComfyUI on the next generation or after a ComfyUI restart.

Downloading & Adding New Models

Method 1 — Download directly on the machine (recommended)

Open a terminal on the desktop (right-click desktop → Open Terminal).
Navigate to the checkpoints folder: 
cd /data/comfyui/models/checkpoints
Download the model with wget — replace the URL with the one from HuggingFace or CivitAI: 
sudo wget -O model-name.safetensors \
  https://huggingface.co/<author>/<repo>/resolve/main/model.safetensors
Refresh ComfyUI — click the Refresh button in the Load Checkpoint node dropdown, or restart ComfyUI: 
sudo docker exec comfyui supervisorctl restart comfyui
Select the model in the Load Checkpoint node dropdown — your new model will appear in the list.

Method 2 — Download via Chrome browser on this machine

Visit civitai.com or huggingface.co in Chrome. Download the .safetensors file. By default downloads land in ~/Downloads — move to the correct folder:

sudo mv ~/Downloads/model-name.safetensors /data/comfyui/models/checkpoints/

Managing the Service

ComfyUI runs as a systemd service and starts automatically on boot. Use the terminal for the commands below.

CommandWhat it does
sudo systemctl status comfyuiCheck if the service is running
sudo systemctl restart comfyuiRestart the service (picks up config changes)
sudo systemctl stop comfyuiStop the service
sudo systemctl start comfyuiStart the service
sudo journalctl -u comfyui -fLive logs for the service
sudo docker logs comfyui --tail 50Last 50 lines of container output
sudo docker exec comfyui nvidia-smiCheck GPU inside the container
sudo docker exec comfyui supervisorctl restart comfyuiRestart only the ComfyUI process, not the whole container
🖼️

Finding Your Generated Images

Every image generated by ComfyUI is automatically saved to:

/data/comfyui/outputs/

Images are organised into subfolders by date. Open the file manager on the desktop and navigate to /data/comfyui/outputs to browse them, or use the terminal:

# List the most recently generated images
ls -lt /data/comfyui/outputs/ | head -20
💡 The outputs folder is outside the container — images are safe even if the container is recreated or updated.
